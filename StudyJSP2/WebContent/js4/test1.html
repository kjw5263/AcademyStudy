form태그 안의 데이터 --> 페이지 전달 방식(get/post)

get 방식 / get method

전송할 데이터를(파라미터) URL의 정보에 붙여서 전송하는 방식
body가 필요 없기 때문에 빠른 전송 속도
데이터의 길이 제약이 존재함 (256byte 초과 불가)
적은 양의 파라미터, 일반적인 데이터 전달 할 때 사용
파라미터가 주소줄(URL)에 모두 보여진다. => 보안적으로 취약
개인정보, 중요 데이터는 전달X
.../itwill.htlm?ID=qwerty&PW=1234

post 방식 / post method

전송할 데이터를(파라미터) URL에 전달X, 본문에 body에 붙여서 전송
body 생성, 많은 양의 데이터를 전달하기 때문에 상대적으로 처리속도가 느리다
파라미터의 길이 제약이 없다 -> 파일 업로드
파라미터가 주소줄(URL)에 전달X -> get방식에 비해 보안적으로 우수하다
개인정보, 중요 데이터를 전달O
.../itwill.html
